Linux 系统调用 int 0x80 与 syscall 详解

int 0x80 是 x86 32位Linux下的软中断指令，用于触发系统调用，进入内核态完成权限操作。

32位使用 int 0x80，寄存器约定：

mov eax, 1    ; 系统调用号
mov ebx, 0    ; 参数
int 0x80      ; 触发中断

64位使用 syscall，寄存器约定：

mov rax, 60   ; 64位系统调用号
mov rdi, 0    ; 参数
syscall       ; 高速系统调用指令

区别：int 0x80 仅兼容32位，64位下推荐使用 syscall，效率更高，支持完整64位地址空间。
